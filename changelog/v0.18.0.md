# v0.18.0 — First standalone release

**Date:** 2026-06-14

The first VardrRunner release cut from its own repository (extracted from the VardrMap
monorepo with full history). This release is lifecycle hardening — no CLI behavior changes.

## Version correctness
- The package previously declared `0.1.0` while already shipping v0.17.x of features
  (daemon, heartbeat, job events, nmap). Since the runner **reports its version to the
  backend on every heartbeat**, the Bridge was being told the wrong number.
- Version is now `0.18.0`, **single-sourced** from `vardrrunner/__init__.py`. `pyproject.toml`
  reads it via setuptools' dynamic-version mechanism, so there is exactly one place to bump.

## Tooling (Engineering Charter §3–4)
- **ruff** replaces `pyflakes` — lint (`E`, `W`, `F`, `I`, `B`, `UP`) plus an enforced
  formatter. Config in `[tool.ruff]`. Cleared all findings (pyupgrade `Optional[X]` → `X | None`,
  import sorting, `raise ... from e` in CLI error handlers). `B008` is ignored as it's the
  required Typer idiom.
- **mypy** added for type checking (`[tool.mypy]`, pragmatic non-strict baseline) — passes clean.
- **Coverage** reporting with a `--cov-fail-under=60` regression gate (currently ~62%).
- CI now runs lint → format-check → mypy → tests+coverage across Python 3.10/3.11/3.12.

## Packaging & docs
- Added a `dev` optional-dependency extra: `pip install -e ".[dev]"` installs pytest, ruff,
  mypy, and type stubs — no more ad-hoc `pip install pytest`.
- Added `LICENSE` (MIT).
- Updated `README.md`, `CLAUDE.md`, and `docs/development.md` to the new toolchain and the
  single-source version/release flow.

## Verification
- 113 tests pass; ruff + ruff-format + mypy all clean.
- No runtime/CLI behavior change.
