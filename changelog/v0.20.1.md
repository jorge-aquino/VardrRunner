# v0.20.1 — Reliability hardening

**Date:** 2026-06-20

Patch release targeting five real-world cracks found during beta use. No new features.

## Recon pagination

`api.VardrMapClient.recon()` previously fetched all results in a single request using the
caller's `limit` as the query parameter. The backend rejected `limit=736` with a 422. The
method now pages in `RECON_PAGE_SIZE` (500) chunks, looping until the requested number of
results are collected or the backend returns a partial page. Large recon sets (httpx,
nuclei, naabu, pipelines) are reliable again.

## Tool failures are now fatal

`runner._run_tool()` raises `ToolError` on any non-zero exit code. All `run_*` functions
now return `None` (they either succeed or raise). A tool that fails — due to a permissions
error, a bad flag, an install issue, or anything else — now marks the job **failed** with
the exit code rather than silently producing no output and drifting into "done". The
`ToolError` is handled by the same `except Exception` block in `commands/jobs.py` that
handles `ToolTimeout`, so no call-site changes were needed.

## `doctor` skips auth on invalid backend URL

`_check_auth()` now calls `config.validate_api_url()` before making any network call. If
the URL is invalid (non-HTTPS remote, malformed scheme, etc.) it returns a `WARN` and
stops — the FAIL already appears in the "backend url" check above it. This prevents the
noisy second-failure that previously showed up when a bad URL was configured.

## Corrupt config handled gracefully

`config.load()` now raises `InvalidConfigFile` (a `ValueError` subclass) instead of a
bare `json.JSONDecodeError` when the config file contains invalid JSON. The error message
includes the file path and a remediation hint. `doctor._collect()` catches it at the top
of the credential/permissions/auth block, emits one clear FAIL check, and continues
running daemon, disk, tool, and pipeline checks — so `doctor` still runs to completion
even when the config file is broken.

## `nmap` version detection fixed

`tool_version()` used `-version` for every tool. `nmap` requires `--version`; with a
single dash it printed its help text which contained no parseable version number. Added
`_VERSION_ARGS` to map each tool to the correct flag. Also broadened the version regex
from `v\d+\.\d+\.\d+` (ProjectDiscovery style) to also match `\d+\.\d+` (nmap style),
so `"Nmap version 7.94 ..."` now resolves to `"7.94"` instead of `"unknown"`.
