# v0.22.0 — Daemon log rotation and timestamps

**Date:** 2026-06-20

Minor release. No breaking changes to the CLI.

## Problem

The daemon's `--log-file` path was opened with `open(log_file, "a", buffering=1)` —
a plain append. On a VPS running a daemon that polls every 5 seconds, this produces
~17,000 lines per day. After a week that's already >100,000 lines with no cleanup.
Additionally, none of the log lines carried timestamps; operators reading a grep hit
had no way to tell when something happened without checking the filesystem's mtime.

## Solution

### `_RotatingLogFile`

New internal class in `commands/daemon.py`. It wraps Python's `RotatingFileHandler`
(5 MB limit, 3 backup files, UTF-8) and implements the file protocol (`write`,
`flush`, `close`) so `rich.console.Console` can use it as its output target.

Key behaviour:
- Buffers incoming chunks until a newline arrives, then flushes each complete line
  to the logger. This ensures timestamps appear once per logical line, not once per
  `Console` chunk.
- At 5 MB the log is rotated to `.log.1`; a new `.log` is opened. After 3 backups
  the oldest is overwritten. Total disk budget: ≈ 20 MB.
- `close()` flushes any remaining buffer and detaches the handler.

### Timestamps

`RotatingFileHandler` is configured with `Formatter("%(asctime)s  %(message)s",
datefmt="%Y-%m-%dT%H:%M:%S")`. Every log line looks like:

```
2026-06-20T14:32:01  Daemon started · PID 12345 · poll 5s · heartbeat 60s
2026-06-20T14:32:06  Cycle complete — 1 job(s) executed.
```

### Rich markup stripped

`Console(file=_log, markup=False)` is used for the log Console so Rich's ANSI escape
sequences don't appear in the log file.

## Tests

5 new tests in `TestRotatingLogFile`:
- `test_writes_timestamped_lines` — timestamp regex present in output
- `test_buffers_partial_writes` — two partial writes merge before flush
- `test_returns_byte_count` — `write()` contract
- `test_no_crash_when_runs_dir_absent` — file created in new directory
- `test_start_writes_to_log_file` — daemon loop output appears in the log file
