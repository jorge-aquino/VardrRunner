# v0.20.0 — Secure credentials + broader recon

**Date:** 2026-06-17

This release gives VardrRunner a strong identity: **trustworthy credential handling** and
**broader recon coverage**.

## Secure credentials
- **OS keychain by default.** `vardrrunner login` stores the API key in the OS keychain
  (macOS Keychain, Windows Credential Locker, Linux Secret Service) instead of plaintext
  config. The backend URL stays in `config.json`. Resolution is
  `VARDRMAP_API_KEY` env > keychain > legacy config file; headless boxes with no keyring
  backend fall back to the (warned) config file. See ADR 0004.
- **`vardrrunner logout`** removes the stored key from the keychain and config file, leaving
  the URL in place (and warns if `VARDRMAP_API_KEY` is still set).
- `doctor` reports the **credential source** without exposing the secret, and only warns
  about config-file permissions when a plaintext key is actually present.

## Broader recon
- **dnsx** (`run dnsx`, `dnsx` job type) — resolve hosts and upload only the **resolvable**
  ones as recon targets, so httpx/nuclei don't waste time on dead names.
- **naabu** (`run naabu`, `naabu` job type) — fast top-ports scan; open ports go to the
  services API (`source: "naabu"`).
- New pipelines: **`deep`** (subfinder → dnsx → httpx → nuclei) and **`ports`**
  (subfinder → dnsx → naabu). Both new tools are picked up automatically by `doctor`,
  `status`, the heartbeat, and pipeline-readiness checks.

## Notes
- `keyring>=24` is a new runtime dependency.
- naabu uploads services with `source="naabu"`; the VardrMap services endpoint must accept
  that source value.

## Tests
- 222 hermetic tests; ~75% coverage. ruff + mypy clean, on Linux/Windows/macOS.
