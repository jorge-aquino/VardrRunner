# v0.19.0 — Capability, safety, and a recon pipeline

**Date:** 2026-06-17

The first feature release cut from the standalone repository. It turns VardrRunner from a
single-tool executor into a fleet-aware, recoverable, diagnosable runner — with typed
contracts, a tool-handler registry, recon pipelines, a real preflight, and release
tooling. Highlights (full detail in `CHANGELOG.md`):

## Features
- **Recon pipelines** — `vardrrunner pipeline run recon` chains subfinder → httpx → nuclei
  (`quick` = subfinder → httpx) in one command; `pipeline list` shows the chains.
- **`vardrrunner doctor`** — deep preflight that exits non-zero on actionable failures with a
  remediation hint per problem (`--json` for scripts). Gates unattended provisioning.
- **`vardrrunner run nmap`** — direct safe-profile service discovery, matching the nmap job.
- **Environment-variable config** — `VARDRMAP_URL` / `VARDRMAP_API_KEY` for containers/CI/VPS.

## Hardening & architecture
- **Per-tool run timeouts** — a hung tool is killed and the job marked failed, never freezing
  the daemon (`VARDRRUNNER_TOOL_TIMEOUT`).
- **HTTPS enforced** for the backend URL — the `vmap_` key is never sent over plain HTTP to a
  non-local host.
- **Resilient API client** — retry + backoff on transient failures (idempotent methods only),
  `User-Agent: vardrrunner/<version>`.
- **Typed, validated configs** + `JobEnvelope` — malformed/drifted payloads fail fast.
- **Tool-handler registry** — `execute_pending_jobs` collapsed into one uniform lifecycle;
  adding a tool is a one-file change (ADR 0002). Direct `run` commands share this path, so
  `run nmap --timing 9` is rejected, not clamped.

## Tooling & release
- ruff (lint + format), mypy, and coverage gate in CI across Python 3.10–3.12 and now
  Linux/Windows/macOS; `pip-audit` dependency scan.
- Tag-driven release workflow: build sdist + wheel, generate a CycloneDX SBOM, attest build
  provenance, publish a GitHub Release, and (opt-in) publish to PyPI via trusted publishing.
  See ADR 0003.
- Packaging metadata for PyPI (authors, classifiers, keywords, project URLs); MIT `LICENSE`.

## Tests
- 196 hermetic tests; ~74% coverage. ruff + mypy clean.
