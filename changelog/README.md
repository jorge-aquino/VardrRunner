# changelog/

Per-version detail notes. The root [`CHANGELOG.md`](../CHANGELOG.md) is the rolled-up,
skimmable index; each file here holds the deeper notes for one release.

## Convention
- One file per release: `vX.Y.Z.md`.
- Add the file in the same commit/PR that bumps the version.
- Summarize the release in `CHANGELOG.md`; put detail (migration notes, rationale,
  command examples) here.
